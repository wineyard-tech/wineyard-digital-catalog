# Part 1: WhatsApp Quote + Zoho Estimate

## 1. CONTEXT

Implement "Get WhatsApp Quote" feature for registered users. When user clicks button in cart, create an Estimate in Zoho Books, store it in Supabase, and send formatted WhatsApp notification.

**User Type:** Registered users only (have `zoho_contact_id` in auth metadata)
**Trigger:** "Get WhatsApp Quote" button in cart page
**Outcome:** Supabase record created → Zoho Estimate created → Supabase record updated → WhatsApp notification sent

**Tech Stack:**
- Zoho Books API (estimate creation)
- Supabase (estimate storage)
- WhatsApp Business API (WABA template: `wineyard_estimate`)

### 3.3 Zoho Estimate Service

Create `/lib/zoho/estimates.ts`

**Function: `createEstimate(contactId, lineItems, userMetadata)`**

**Input:**
```typescript
{
  contactId: string,           // Zoho contact ID
  lineItems: Array<{
    item_id: string,           // Zoho item ID
    name: string,
    quantity: number,
    rate: number,              // Unit price
    item_total: number         // quantity * rate
  }>,
  userMetadata: {
    contact_name: string,
    company_name: string
  }
}
```

**Zoho API Call:**
```
POST https://www.zohoapis.in/books/v3/estimates
Headers: Authorization: Zoho-oauthtoken {token}
Query: organization_id={org_id}

Body:
{
  customer_id: contactId,
  estimate_number: (auto-generated by Zoho),
  reference_number: (optional),
  date: YYYY-MM-DD (today),
  expiry_date: YYYY-MM-DD (today + 1 day),
  line_items: [...],
  notes: "Quote requested via WineYard Digital Catalog",
  zoho_sync_status: "sent"
}
```

**Return:**
```typescript
{
  zoho_estimate_id: string,
  estimate_number: string,      // EST-XXXXXX from Zoho
  total: number,
  line_items: array
}
```

**Error Handling:**
- Retry once on network failure
- Log all Zoho API errors
- Return structured error for UI display

### 3.5 API Route

Create `/app/api/quotes/send/route.ts` (POST)

**Request Body:**
```typescript
{
  cartItems: Array<{
    zoho_item_id: string,
    name: string,
    quantity: number,
    rate: number
  }>
}
```

**Logic:**
1. Verify user is authenticated (get session)
2. Extract `zoho_contact_id` from user metadata
3. If missing → return 403 "Registration required"
4. Calculate totals from cart items
5. **Generate cart hash** (hash of sorted line_items JSON for duplicate detection)
6. **Check for duplicate:** Query Supabase for estimate with same `cart_hash` + `user_id` + `created_at > now() - 24 hours` + `zoho_sync_status != 'failed'`
   - If found: Re-send WhatsApp with existing estimate (skip Zoho creation)
   - If not found: Continue to step 7
7. **Create Supabase record FIRST:**
   Get the generated `id` (uuid) from Supabase response
8. **Call Zoho API** to create estimate (zoho_sync_status: sent)
   - **If Zoho API fails:** Retry once after 2 seconds
   - **If still fails after retry:**
     - Update Supabase record:
       - `zoho_sync_attempts = 1`
       - `zoho_sync_error = {error message}`
       - `zoho_sync_status = 'pending_zoho_sync'` (unchanged)
     - Return to user: `{success: false, error: "Quote request saved but sync failed. We'll retry shortly.", estimateId: {supabase_id}}`
     - Background job will retry later
9. **If Zoho API succeeds:**
   - Update Supabase record:
     - `zoho_estimate_id = {from Zoho response}`
     - `estimate_number = {EST-XXXXXX from Zoho}`
     - `zoho_sync_status = 'sent'`
     - `zoho_sync_attempts = 1`
10. Format line items for WhatsApp template
11. Generate deep link: `{APP_URL}/cart?estimate_id={supabase_id}` (use Supabase ID, not Zoho ID)
12. Send WhatsApp notification
13. Update estimate record: `whatsapp_sent = true`, `whatsapp_message_id = {id}`
14. Return: `{success: true, estimateNumber: "EST-XXXXXX", estimateId: {supabase_id}}`

**Response:**
- 200: Estimate created and sent (or sync pending with retry scheduled)
- 400: Invalid cart data
- 403: User not registered
- 500: Unrecoverable error

### 3.8 Background Retry Job (Optional - Can Defer to Phase 2)

Create `/lib/cron/retry-zoho-estimates.ts` or use Vercel Cron

**Schedule:** Every 5 minutes

**Logic:**
1. Query Supabase for estimates where:
   - `zoho_sync_status = 'pending_zoho_sync'`
   - `created_at > now() - 24 hours`
   - `zoho_sync_attempts < 10`
2. For each estimate:
   - Increment `zoho_sync_attempts`
   - Try Zoho API to create estimate
   - **If success:**
     - Update with Zoho IDs (`zoho_estimate_id`, `zoho_estimate_number`)
     - Change `zoho_sync_status = 'sent'`
     - Send WhatsApp notification (if not already sent)
   - **If fail:**
     - Update `zoho_sync_error` with latest error
     - Keep `zoho_sync_status = 'pending_zoho_sync'`
     - Will retry in next cron cycle
3. For estimates older than 24 hours with `zoho_sync_status = 'pending_zoho_sync'`:
   - Change `zoho_sync_status = 'failed'`
   - Send alert to admin WhatsApp: "Estimate sync failed after 24hrs: Estimate ID {id}, User: {phone}"

## NOTES

- **CRITICAL: Create Supabase record FIRST, then Zoho.** This enables retry capability.
- Zoho estimate number is auto-generated by Zoho, do not create manually
- Valid until = created_at + 24 hours (show in WhatsApp message)
- Cart hash prevents duplicate estimates for identical carts within 24 hours
- Use Supabase `id` (uuid) for deep links, not Zoho ID (since Zoho ID may be NULL during pending sync)
- WhatsApp only sent after successful Zoho sync (zoho_sync_status = 'sent')
- If Zoho sync pending, user can still see estimate in app but WhatsApp not sent yet
- Background retry job is optional for MVP - can implement in Phase 2
- WhatsApp template must be pre-approved in WABA dashboard before testing
- Test with real Zoho sandbox/test organization first
